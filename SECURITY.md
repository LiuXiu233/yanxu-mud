# 安全政策

## 支持版本

正式发布后，最新 1.x 次版本获得安全修复；旧次版本在新次版本发布后保留 90 天安全支持。0.x 为开发版本，不承诺生产安全支持。

## 报告漏洞

请使用 GitHub 仓库的私密漏洞报告功能，不要公开披露可利用细节。报告应包含受影响版本、重现条件、影响范围和可行的缓解措施。维护者会在 72 小时内确认收到，并在验证后协调修复与披露。

## 生产边界

Telnet 不提供传输加密，只适合受信网络或 TLS 终止代理之后。HTTP/WebSocket 的生产部署必须使用反向代理提供 TLS、请求限制和来源策略。内容包默认不受信任：言据不执行代码，代码插件则拥有宿主进程权限，必须经过人工审查。

密码、令牌、Cookie、Authorization、数据库凭据和私钥不得写入日志。默认配置只监听回环地址，管理接口和热重载要求独立权限与审计记录。

账户服务不实现密码学算法，只保存注入适配器返回的凭据信封。生产适配器必须使用成熟的 Argon2id、scrypt 或 bcrypt 库，为每个密码生成独立随机盐，配置可升级的工作因子，并以恒时方式验证。仓库测试中的 SHA-256 适配器是确定性测试替身，不得用于生产。接入层还必须对不存在账户和错误密码采用一致的外部响应、统一计时策略与组合限流；应用层返回值、事实事件和日志不得包含密码或凭据摘要。

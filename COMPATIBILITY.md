# 兼容政策

## 版本范围

言域使用语义化版本。1.x 中：

- 补丁版本只修复缺陷或补充文档，不改变稳定协议语义；
- 次版本可以添加向后兼容的 API、组件、系统、消息节点和内容字段；
- 删除或改变稳定 API、存档格式、内容 Schema、插件协议、错误代码语义，只能进入新的主版本。

## 1.0 稳定面

1.0 将冻结实体组件、命令、领域事件、内容包、结构化消息、状态仓储六项协议，以及存档、事件日志、插件和错误代码格式。未知组件和自定义消息节点必须往返保留，以支持前向兼容。

0.3.0 首次公开内容包、内容文档、内容 Schema、内容图、制品和热重载格式 v1。这些格式在 0.x 中仍是预览面；内容作者应显式声明引擎约束，并在升级前运行内容检查和内容差异。

0.4.0 首次公开言据文件世代、SQLite 世代、事件日志和快照恢复报告格式 v1，并继续使用状态仓储与世界快照格式 v2。数据迁移历史要求严格前缀和定义摘要。以上持久化格式在 0.x 中仍是预览面；升级前必须创建快照、运行迁移计划并保留原数据库或完整世代目录。

0.5.0 开发周期公开账户、角色、权限、空间、物品、装备、属性、状态、冷却、技能、战斗、死亡、任务与对话服务。技能定义通过稳定行为编号引用代码，任务和对话定义保留在内容图，动态组件只保存运行进度；这些玩法 API 与错误代码在 0.x 中仍是预览面。

## 工具链

- 最低言序版本：1.1.12；支持兼容的 1.x。
- 包清单和锁文件格式：2。
- 言据格式：1；内容包 Schema 版本与引擎版本独立。
- 命令行以当前工具帮助列出的语法为准。言序 1.1.12 未提供 `run` 英文别名；仓库验证使用 `yanxu <文卷> -- ...`、`yanxu 字节 <文卷> -- ...` 和 `yanxu 包 运行 <目录> -- ...`。大型内容构建和基准使用全局 `--max-steps` 显式提高单次执行预算。

不受支持的编译器或平台会给出显式诊断。平台专属网络或数据库测试必须以条件跳过记录原因，不能静默通过。

## 弃用

稳定 API 至少保留一个次版本的弃用期。弃用项必须出现在静态检查诊断、变更记录和迁移指南中，并提供等价替代路径。

# 命令协议

命令协议位于 `src/命令/输入解析.yx` 和 `src/命令/命令协议.yx`。输入层只把文字变成可序列化调用；执行层只返回结构化结果，不持有连接，也不向控制台、Telnet 或 WebSocket 写文字。

## 调用格式

`输入解析.解析` 支持空格与全角空格、单双引号、中文直角/弯引号、空引号参数和反斜线转义。首个词元是命令名，其余词元保持为位置参数：

```yanxu
定 调用：典 为 输入.解析（"告诉 子衿 \"今晚 青石桥见\""）；
```

结果使用 `言域/命令调用` v1 信封，含原文、非空名称和文字位置参数。未闭合引号、末尾孤立转义、空输入、空引号命令名和畸形手工调用均返回 `YANYU_COMMAND_PARSE`。

## 定义和匹配

```yanxu
定 查看 为 命令.新定义（{
    「编号」：「青石镇:命令/观察」，
    「名称」：「观察」，
    「别名」：【「look」，「l」】，
    「分组」：「游戏」，
    「优先级」：100，
    「权限」：【「世界.观察」】，
    「冷却」：500
}，处理观察）；
```

中文名称原样匹配，英文别名按 Unicode 小写形式匹配。名称、别名、分组和权限项不得含首尾空白。多个定义共享名称或别名时，优先级数值较小者先匹配；相同优先级按稳定编号排序。上下文可用 `命令组` 限定匹配空间。注册表支持注册、替换、分组查询和只含普通值的元数据清单。

别名和权限在定义内部保存，公共读取只返回副本；调用方不能通过修改可变列绕过权限或改变已经注册的匹配规则。定义一经注册或替换即冻结，不能再原地追加前置条件或命令中间件；热更新须建立新定义并调用 `替换`，使行为切换保持原子。

## 执行上下文和授权

上下文是普通值典，常用字段包括主体编号、账户编号、会话编号、权限、命令组和世界时间。执行器深复制上下文与调用，随后附加所选命令元数据；中间件修改只影响本次执行，不会反向修改调用方的数据。

权限采用“全部满足”语义，`*` 表示明确的全权限。权限检查先于前置条件和处理器。前置条件只能返回逻辑值或结构化命令结果；空值、数值和畸形典按失败处理，避免遗漏 `归 真` 时意外放行。

冷却按 `主体编号 → 账户编号 → 会话编号` 的顺序隔离，并为三类编号加不同前缀以防碰撞。带冷却的命令必须有至少一个非空身份编号，并使用注入的世界时钟或显式 `世界时间`；两者都必须是有限非负数，与世界时钟边界一致地允许小数毫秒。时钟倒退会以上下文错误拒绝。只有规范化后的成功结果才记录冷却；权限、前置条件、处理器或结果验证失败不会消耗冷却。

同一执行器会在处理链开始前以冷却键和命令编号的二层映射占位，同步重入以 `YANYU_COMMAND_REENTRANT` 拒绝，因而不能在首次冷却尚未记录时递归绕过。到期记录在下次检查时惰性删除；会话断开、角色卸载或热替换命令时应调用 `清理主体冷却`，全量重置可调用 `清理全部冷却`，避免长服保留临时身份。所有命令另有默认 32 层、可通过 `设置最大执行深度` 收紧的嵌套预算；该值必须是有限正整数，超限使用 `YANYU_COMMAND_RECURSION_LIMIT`。

## 中间件和结果

执行器级中间件包裹命令级中间件，注册顺序即进入顺序，返回时逆序退出。中间件接收上下文、调用和下一步回调，可做追踪、限流或事务包装。同一个中间件的下一步回调只能在本次中间件调用期内同步调用一次；重复或延迟调用都以 `YANYU_COMMAND_MIDDLEWARE` 稳定失败，避免处理器或世界事务重复、越期执行。上下文中的 `调用` 与处理器实参指向同一份已经隔离的工作副本，规范化修改不会出现双重视图。

全局中间件只包裹已经通过解析、查找、权限、冷却和前置条件检查的处理链；未知命令以及这些前置阶段的拒绝不会进入中间件。若要记录所有分派尝试，应在应用入口或后续审计子系统中记录安全摘要，不应把全局中间件当成完整审计观察层。调用信封中的 `原文` 和位置参数可能含密码等敏感数据，日志默认不得记录它们，除非命令显式声明并执行了字段级脱敏。

结果使用 `言域/命令结果` v1，固定包含成功、命令编号、数据、消息和错误。成功消息中的每一项都必须是通过消息协议验证的完整信封，而不是裸节点或文字；失败错误含稳定代码、中文消息和详情典。所有数据必须是无环且处于复杂度预算内的普通可序列化值，否则转换为 `YANYU_COMMAND_RESULT`；处理链抛错转换为 `YANYU_COMMAND_HANDLER`。

稳定错误码还包括定义无效、编号重复、未知命令、权限不足、前置失败、冷却和上下文错误。调用方应按代码分支，中文消息只用于诊断和显示。

面向调用方的解析与回调错误只返回稳定代码和安全类别，不回显原始输入、内部异常消息或源码绝对路径；完整诊断只能进入受控且脱敏的服务日志。

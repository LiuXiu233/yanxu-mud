# 言域 0.4.1 公共 API

本文是 0.4.1 世界核心、内容运行时与持久化的言包导入索引。语义和不变式由各专题文档定义；源码中以 `公` 声明的类、协议、函数和错误常量均可从对应子模块使用。

0.x 是预览系列；公共 API、协议版本和存档格式到 1.0 才冻结。具体游戏不应导入 `src/` 内部路径，而应使用下表中的言包路径。

```yanxu

公 定 名称：文；
公 定 英文标识：文；
公 定 版本：文；
公 定 最低言序版本：文；
公 定 开发阶段：文；
公 法 服务器信息（）：典；
```

`0.4.1` 的 `服务器信息` 返回名称、英文标识、版本、最低言序版本、开发阶段和状态；开发阶段为“持久化”，状态为“预览”。

## 账户

`账户` 是 0.5.0 开发周期新增的应用层模块：

```yanxu
公 法 默认配置（）：典；
公 法 公开账户（账户实体）：典；
公 类 账户服务；
```

`账户服务（仓储，生成摘要，验证摘要，配置）` 要求调用方注入两个密码适配函数。`生成摘要（密码）` 返回至少含 `方案` 与 `摘要` 的普通凭据典；`验证摘要（密码，凭据）` 返回逻辑值。引擎不提供自制密码算法，生产部署必须注入支持独立盐、工作因子升级和恒时验证的成熟 Argon2id、scrypt 或 bcrypt 实现。测试中的确定性 SHA-256 适配器只用于验证协议，禁止用于真实账户。

实例提供：

```yanxu
创建账户（账户编号，登录名，密码，上下文）：典；
读取账户（账户编号）；
验证登录（登录名，密码，上下文）：典；
设置冻结（账户编号，冻结，上下文）：典；
修改密码（账户编号，旧密码，新密码，上下文）：典；
```

上下文至少给出稳定且单次使用的 `事务编号`；可携带世界/现实时间、追踪编号、命令编号、内容包版本、操作者编号与权限。登录名以大小写不敏感键建立原子世界变量索引。账户实体、索引、失败计数、锁定或冻结状态在同一世界事务中提交；账户创建、登录、拒绝、冻结变更与改密均产生不含凭据的事实事件。`设置冻结` 需要 `言域:权限/账户管理`。`公开账户`、成功结果、结构化错误和消息均不含密码、方案参数或摘要。

公开错误常量区分参数无效、账户重复/缺失、凭据无效、登录受限、账户冻结、权限不足和密码适配器故障。登录名不存在与密码错误都返回 `凭据无效错误`，接入层仍须实施统一响应时间、IP/账户组合限流与审计。

## 角色

`角色` 模块导出 `角色服务`、`默认配置`、`公开角色`、账户/角色组件编号、角色管理权限和稳定错误常量。服务构造器为 `角色服务（仓储，配置）`，实例提供：

```yanxu
创建角色（账户编号，角色编号，名称，模板，上下文）：典；
读取角色（角色编号）；
列出角色（账户编号）：列；
请求删除（账户编号，角色编号，上下文）：典；
取消删除（账户编号，角色编号，上下文）：典；
确认删除（账户编号，角色编号，上下文）：典；
```

角色模板至少包含稳定 `初始房间`，可覆盖原型编号、描述、基础属性、最大生命和背包容量。创建操作在一个世界事务中检查账户状态、角色上限、编号和同账户名称唯一性，写入账户反向索引，并建立含角色、账户、描述、位置、可移动、背包、装备、属性、状态、战斗、技能、冷却、对话、任务、权限、频道和持久化组件的玩家角色实体。所有组件使用 v1 信封，初始快照修订为 0；事实事件和结构化消息只在仓储提交成功后出站。

删除是三步状态机。`请求删除` 写入可注入世界时间上的保护截止点，`取消删除` 恢复正常状态，`确认删除` 只在截止时间到达后于同一事务删除实体并从账户角色列移除编号。账户本人可以管理自己的角色；携带 `言域:权限/角色管理` 的操作者可以跨账户管理，但不能跳过保护期。服务拒绝悬空账户索引、跨账户角色、重复名称、数量超限和非法状态。

## 权限

`权限` 模块导出内置角色组与权限编号、`默认角色定义`、`规范角色定义`、`有效权限`、`构建命令上下文`、`权限服务` 和 `建立系统`。权限服务提供权威查询、命令上下文投影、首次管理员引导、显式权限授予/撤销和角色组授予/撤销。成功变更在同一世界事务中提交实体状态、引导变量、事实事件和结构化消息。

管理员角色组包含游戏、管理、账户管理、角色管理、权限管理以及移动、物品、装备、属性、状态、冷却、技能、战斗、死亡、任务、对话、掉落、时间、刷新和 AI 管理权限。首次引导只能由含自归属账户组件的目标账户执行一次，角色实体不能伪装引导；后续轮换必须先建立第二个管理员，服务禁止撤销末位管理员。`建立系统` 返回 `言域:系统/权限`，支持查询、投影和全部变更动作。完整不变式、事件和安全边界见[权限模型](PERMISSIONS.md)。

## 实体与组件

`实体` 模块的主要声明：

```yanxu
公 类 实体；
公 法 新实体（编号：文，原型编号：文）：实体；
公 法 序列化实体（实体值：实体）：典；
公 法 恢复实体（快照）：实体；
公 法 尝试恢复实体（快照）：典；
公 法 合并组件（基础：典，覆盖：典）：典；
公 法 从原型快照（编号：文，原型快照：典，动态组件：典，动态标签：列）：实体；
公 法 应用变更集（实体值：实体，变更集：典）：实体；
公 法 尝试应用变更集（实体值：实体，变更集：典）：典；
```

`实体` 实例提供原型、修订、组件、标签、快照、变更集和确认变更操作。公开错误常量包括 `编号非法错误`、`组件已存在错误`、`组件不存在错误`、`快照无效错误`、`变更集无效错误` 和 `修订冲突错误`。

`组件` 模块导出 `组件定义` 和 `注册表` 两个类。定义可取默认值、检查数据、建立新信封、规范旧信封、复制和转换为典；注册表可注册、替换、查询、规范与列出定义。错误常量为 `定义重复错误`、`定义不存在错误`、`定义无效错误`、`数据无效错误` 和 `迁移失败错误`。

`通用组件.建立注册表（）` 返回已注册 24 种 1 版通用组件的新注册表。

## 系统

```yanxu
公 类 系统；
公 类 注册表；
```

`系统` 支持启用、禁用、启动、停止、处理和装饰；`注册表` 支持注册、运行中原子替换、查询、启停、装饰、确定性排序与批量生命周期。错误常量为 `系统重复错误`、`系统不存在错误` 和 `生命周期错误`。注册表本身不内置玩法规则，具体玩法系统由空间、物品、装备、属性、状态、技能、战斗、死亡、任务和对话等公共子模块提供。

## 空间

`空间` 模块导出方向规范、房间规范、内容图解析器、`空间服务`、`建立移动系统` 和 `建立观察系统`。房间名称、描述和出口来自静态内容，不写入状态仓储；实体位置、可移动与战斗状态来自动态组件。

`空间服务.移动` 在一个世界事务中校验操作者、当前房间、方向出口、目标房间、可移动标记和战斗状态，更新位置组件，并提交 `言域:事件/实体移动` 与目标房间场景消息。`观察` 只读查询同房间动态实体，按标签生成物品或人物引用，并返回场景描述和房间出口。移动系统编号为 `言域:系统/移动`、优先级 100；观察系统编号为 `言域:系统/观察`、优先级 110，因此同一注册表轮次会先移动再观察。详细约束见[系统与调度](SYSTEMS.md)。

## 物品与容器

`物品` 模块导出：

```yanxu
公 法 规范物品定义（编号：文，原值）：典；
公 法 内容图物品解析器（内容图）；
公 法 读取物品定义（解析器，编号：文）：典；
公 类 物品服务；
公 法 建立物品系统（服务：物品服务）；
公 法 建立背包系统（服务：物品服务）；
公 法 建立容器系统（服务：物品服务）；
```

`物品服务（仓储，定义解析器）` 通过注入解析器读取静态物品定义，不把名称、描述、类型、装备槽、属性、价值和重量复制为权威动态配置。实例提供 `创建物品`、`读取物品`、`拾取`、`放下`、`转移` 和 `查看容器`。物品实体保存定义编号、数量、绑定状态、创建时间和位置；读取投影把当前静态定义与动态状态组合为普通典。

创建和转移在世界事务中同时维护物品位置与背包/容器编号列；容量不足、来源错配、悬空引用或提交冲突会回滚全部改动。容量 `0` 表示不设项目数上限，正数按实例数限制。不存在对应动态实体的稳定位置可作为静态地面位置；若位置实体具有背包或容器组件，其双向引用必须一致。容器查看会验证每个实例仍存在且位置匹配，并返回结构化物品投影和消息。

创建需要 `言域:权限/物品管理`；拾取和放下默认只允许角色本人；容器间转移只允许来源控制者或物品管理员。权限列必须来自 `包:言域/权限` 的权威投影，不能直接信任客户端。成功写操作产生 `言域:事件/物品创建` 或 `言域:事件/物品转移` 事实事件。三个系统编号依次为 `言域:系统/物品`、`言域:系统/背包`、`言域:系统/容器`，优先级为 120、130、140。公开错误常量区分参数、定义、实例、容器、容量、位置、权限、移动限制和系统动作错误。

## 属性、装备与状态

`属性` 模块导出 `规范属性值`、`计算当前属性`、`读取属性`、`应用属性修正`、`应用聚合属性`、`修正源聚合器`、`新修正源聚合器`、`属性服务` 和 `建立属性系统`。修正源形如 `{来源，属性}`，来源必须是唯一稳定编号；聚合器按稳定提供器编号调用各权威来源，再按来源编号确定性排序并对每个有限数属性做加法。提供器支持注册、替换、移除和查询编号列，跨提供器重复来源会拒绝。属性组件只保留基础、当前和来源编号列，不持久化静态装备或状态定义副本。若实体带战斗组件，重算会同步攻击、防御和最大生命；攻击与防御下限为零，最大生命必须为正，降低上限时当前生命会被截断到新上限。

`属性服务（仓储，聚合器）.重算（实体编号，上下文）` 需要 `言域:权限/属性管理`，在一个世界事务中从聚合器重新解析全部来源并提交属性、战斗投影、`言域:事件/属性重算` 和结构化消息。`应用属性修正` 与 `应用聚合属性` 是供装备、状态等可信规则在其既有事务工作实体上复用的领域变更边界。属性系统编号为 `言域:系统/属性`，优先级 160。

`装备` 模块导出 `建立装备属性提供器`、`建立装备属性聚合器`、`装备服务` 和 `建立装备系统`。构造器为 `装备服务（仓储，物品服务，属性聚合器）`，实例提供 `穿戴（角色编号，物品编号，槽位，上下文）`、`卸下（角色编号，槽位，上下文）` 和 `查看（角色编号）`。穿戴要求物品由角色持有、位于背包编号列且静态定义声明目标 `装备槽`；成功后物品位置仍指向角色，只从背包引用移到装备槽引用。卸下执行相反变更并受背包容量限制。

背包、槽位、属性和战斗投影在同一世界事务中更新；槽位占用、不兼容、背包已满、权限或提交失败不会留下半穿戴状态。角色本人可管理自己的装备，跨角色操作需要 `言域:权限/装备管理`。成功操作产生 `言域:事件/装备变更` 与结构化消息。装备系统编号为 `言域:系统/装备`，优先级 150，因此注册表默认先处理装备动作，再进入独立属性系统阶段。

`状态` 模块导出状态定义规范与解析、状态列规范、施加/移除/到期计算、`建立状态属性提供器`、`注册状态属性提供器`、`写状态并重算`、`状态服务` 和 `建立状态系统`。静态定义包含名称、描述、每层属性、持续毫秒、最大层数和 `刷新`、`叠加` 或 `拒绝` 策略；动态组件只保存稳定状态实例、定义、来源、层数、开始与到期时间。到期时间 `0` 表示永久，其他状态在 `当前时间 >= 到期时间` 时失效。

`状态服务（仓储，定义解析器，属性聚合器）` 提供读取、受 `言域:权限/状态管理` 保护的施加、移除和到期清理。每次写操作在同一世界事务中更新状态组件与全部属性来源，产生 `言域:事件/状态变更` 和结构化消息；没有到期项目的清理保持修订不变。状态系统编号为 `言域:系统/状态`，优先级 170。装备和状态服务必须共享一个同时注册两类提供器的聚合器，确保任一子系统重算时保留另一类有效来源。

`战斗` 模块导出战斗数据与伤害参数规范、`计算伤害`、`执行伤害`、`建立伤害技能行为`、`战斗服务` 和 `建立战斗系统`。基础公式为 `max(最低伤害, floor(攻击 * 倍率 / 100) + 固定伤害 - 防御)`，实际伤害不超过目标当前生命。攻击要求不同且存活的双方位于同一位置；跨实体代用需要 `言域:权限/战斗管理`。双方投影、战斗消息、`言域:事件/战斗伤害` 以及致死时的死亡标签、死亡时间、击败者编号和 `言域:事件/实体死亡` 同事务提交。内置行为编号为 `言域:技能行为/伤害`，系统编号为 `言域:系统/战斗`，优先级 200。

`死亡` 模块导出 `是否死亡`、`复活实体`、`死亡服务` 和 `建立死亡系统`。本人可复活自己；跨实体复活需要 `言域:权限/死亡管理`。恢复生命为零表示恢复至最大生命，正值会截断；复活房间为空时保留现有位置，非空时要求位置组件。服务把战斗状态、死亡元数据、标签、可选位置、`言域:事件/实体复活` 和结构化消息原子提交。系统编号为 `言域:系统/死亡`，优先级 210。

`时间服务（仓储，时钟，行为注册）` 可直接作为世界循环时钟，其 `调度器（）` 返回同仓储适配器。计划、取消和执行需要 `言域:权限/时间管理`。`计划到事务` 与 `取消到事务` 是供可信领域服务组合已有世界事务的无权限内部边界；它们只改计划任务和单调时间，不自行提交、发布事件或发送消息。执行会重新读取任务并核对候选修订与到期时间，在一个世界事务内运行行为、提交行为事件/消息、删除一次性任务或按 `原到期 + 间隔` 推进重复任务，再发布 `言域:事件/计划任务执行`。行为、消息、确认或仓储任一失败会回滚，原任务下一轮仍可重试；成功后旧候选不能重复执行。

注入时钟必须返回有限非负整数毫秒。每次写操作把当前值保存到 `言域.时间.最后毫秒`，读取和写入都拒绝低于该值的时钟，避免重启或错误适配器让世界时间倒退。`建立任务处理器` 把执行结果的事实事件与结构化消息接入世界循环。时间系统编号为 `言域:系统/时间`，优先级 40。

`SQLite仓储` 导出 `SQLite仓储`、`SQLite事务`、`验证世代`、`以连接`、`以执行器`、`打开为` 和 `打开`。`以执行器` 通过言舟的注入边界运行协议测试；`打开为` 可指定 SQLite CLI 路径和超时。实例完整实现状态仓储协议与事件日志扩展，并提供关闭状态、世代和恢复报告。SQLite 最低版本为 3.38.0。

`数据迁移` 导出 `数据迁移`、`新迁移`、`简单迁移`、`计划`、`计划摘要` 和 `执行`，以及定义无效、历史分歧、执行失败和检查失败四个稳定错误码。迁移按调用方给出的稳定顺序执行，已应用历史必须是当前定义的严格前缀；编号、版本或定义摘要变化会返回历史分歧。一个调用中的全部待办迁移、后置检查和历史记录共享同一仓储事务，任一失败会回滚整个批次，并保留可重试的事务冲突错误。

`世界事务` 导出 `世界事务` 类和 `开始（状态仓储，编号）`。实例转发仓储事务的状态操作，并提供 `修改实体`、`暂存事件`、`暂存消息`、`待提交计数`、`提交`、`回滚` 和 `已提交输出`。

## 世界循环与虚拟时间

`虚拟时钟` 导出：

## Schema、文档、引用与原型

`内容Schema` 导出规则验证、定义规范和 `注册表`。注册表提供注册、替换、查询、版本、适用类别、编号列、检查、验证和断验；规则字段与演进边界见 [CONTENT_SCHEMA.md](CONTENT_SCHEMA.md)。

`内容文档` 导出内容编号、类别和本地化键检查，文档规则、规范、文字解析和 `定义注册表`。注册表支持单项或批量注册、替换、按包或类别列举，以及查询引用概要。批量加载路径接受已经由文件加载器验证过归属的规范文档。

`内容引用.建立图` 和 `建立已排序图` 验证包可见性、目标存在、类别声明和原型循环；`尝试建立图` 返回公开错误而不抛出。

`内容原型.注册表` 接受定义注册表，提供 `解析`、`解析全部`、`解析索引`、缓存查询和清理。结果包含从根到叶的原型链与深度，以及合并后仍需 Schema 校验的普通数据。

## 本地化与内容图

`本地化` 导出资源规范、插值、复数类别和 `目录`。目录按默认语言、请求语言、回退语言和内容包加载顺序查找，可批量检测缺失键；`翻译复数` 预留复数类别接口。插值只替换调用方显式提供的参数，缺失或残留占位符会失败。

`内容图.构建` 接受清单、Schema、文档、本地化、语言和行为编号；`构建已加载` 用于可信文件加载边界；`尝试构建` 返回结果信封。`内容图` 实例提供包、Schema、原始内容、已解析内容、引用图、本地化、行为、统计、分项指纹和总摘要查询。所有返回值均保持隔离，摘要对相同语义输入确定。

## 值与错误边界

世界状态、事件载荷、命令结果和结构化消息中只允许无环、处于复杂度预算内的普通言序值。数值必须有限；修订、序号、版本和预算使用协议指定的非负或正整数。公共错误以稳定 `YANYU_*` 代码与中文消息暴露，不应依赖运行时原始踪迹或源码路径。

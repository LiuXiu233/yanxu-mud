# 内容包

内容包是言域静态世界的发布、依赖和信任边界。每个包是一个独立目录，根部必须有 `内容包.yj`；房间、物品、NPC、任务、对话和其他静态定义位于清单列出的 `.yj` 文件中。内容文件只承载普通数据，不执行任意代码。

## 目录约定

```text
我的世界/
├── 内容包.yj
├── Schema/
│   └── 房间.yj
├── 内容/
│   ├── 房间.yj
│   └── 物品.yj
└── 本地化/
    └── zh-CN.yj
```

清单中的路径必须是规范相对路径：使用 `/`，不能是绝对路径，不能包含 `.`、`..`、反斜杠、空段或首尾空白，且必须以 `.yj` 结尾。清单、Schema、内容和本地化路径在同一包内不可冲突。

## 清单协议

协议名为 `言域/内容包`，格式版本为 `1`。清单必须恰好包含下列字段：

| 字段 | 类型 | 语义 |
| --- | --- | --- |
| `协议` | 文 | 固定为 `言域/内容包` |
| `格式版本` | 数 | 固定为 `1` |
| `包编号` | 文 | 稳定命名空间，最长 64 字符 |
| `包名称` | 文 | 面向维护者的名称 |
| `版本` | 文 | 语义版本 |
| `引擎版本` | 文 | 言域版本约束 |
| `依赖` | 列 | 必需依赖及其版本约束 |
| `可选依赖` | 列 | 安装后可见、未安装不报错的依赖 |
| `加载顺序` | 数 | 同层依赖的确定性排序键 |
| `入口` | 列 | 内容文档或内容文档集路径 |
| `Schema` | 列 | 内容 Schema 路径 |
| `本地化` | 据 | 语言到资源文件列的映射 |
| `迁移` | 列 | 预留的内容迁移声明 |
| `权限声明` | 列 | 内容需要的稳定权限编号 |
| `内容签名` | 据 | 签名元数据预留字段，不代表当前自动信任 |
| `热重载` | 理 | 是否允许进入热重载策略评估 |
| `扩展` | 据 | 归属包自行解释的普通数据 |

依赖项形如 `据【「包编号」：「基础世界」，「版本」：「^1.2.0」】`。支持精确版本、比较式、空格相交约束、`*`、`^` 和 `~`。加载器先解析全部包头和依赖图，再按依赖、`加载顺序`、包编号稳定排序；缺失依赖、版本不符、循环或重复包编号均在读取正文前失败。

## 内容文档

单文档协议为 `言域/内容文档`，编号固定采用：

```text
包编号:类别/标识
```

完整文档包含 `协议`、`格式版本`、`包编号`、`类别`、`编号`、`Schema编号`、`Schema版本`、`数据`、`原型`、`引用`、`行为编号`、`本地化键` 和 `扩展`。`引用` 的键是数据路径，例如 `$.出口.北`；值是目标内容编号或带类别、可选标记的引用说明。

同类别内容可以写成 `言域/内容文档集`，共享 `包编号`、`类别`、`Schema编号` 和 `Schema版本`。每个项目用 `标识` 生成完整编号，并可声明 `数据`、`原型`、`引用`、`行为编号`、`本地化键` 和 `扩展`。加载后两种形式具有相同语义。

## 可见性与行为

内容只能引用本包、必需依赖或已安装的可选依赖。引用解析会检查目标存在、声明类别与实际类别一致，并检测跨包原型循环。可选引用可以缺失，但仍会进入内容图的“缺失可选引用”统计。

言据不得包含可执行回调。需要逻辑的内容通过稳定行为编号声明依赖，宿主用言序代码注册编号；构建内容图时，未注册的必需行为会以 `YANYU_CONTENT_BEHAVIOR_MISSING` 失败。

## 装载预算

默认文件边界是：

- 清单不超过 1 MiB；
- 单个项目文件不超过 4 MiB；
- 一个内容世界最多 4096 个文件、总计 64 MiB；
- 单文件最多 4096 个 Schema、文档或本地化资源项目；
- 一次加载 1 至 256 个包根目录。

加载器在读取前后核对文件元数据，检测读取期间被替换的文件。错误只暴露稳定代码和已清理的来源信息，不回传原始系统错误或任意文件内容。

## CLI

检查一个或多个内容包：

```bash
yanxu tools/言域.yx -- 内容检查 examples/青石镇/内容
```

构建内容世界制品：

```bash
yanxu --max-steps 20000000 tools/言域.yx -- 构建 --输出 世界.yj 包甲 包乙
```

构建单包发布制品：

```bash
yanxu --max-steps 20000000 tools/言域.yx -- 内容打包 --目标包 包乙 --输出 包乙.yj 包甲 包乙
```

比较两个已验证内容世界：

```bash
yanxu tools/言域.yx -- 内容差异 --旧包 旧甲 --旧包 旧乙 --新包 新甲 --新包 新乙
```

这些命令均支持 `--引擎版本`、`--默认语言`、重复的 `--回退语言`、重复的 `--行为` 和 `--json`。`--max-steps` 是言序全局旗标，必须放在文卷路径前；示例预算覆盖青石镇完整制品的多次规范序列化和回读校验。构建命令默认拒绝覆盖已有文件，显式传入 `--覆盖` 才允许覆盖。

## 制品

世界制品协议为 `言域/内容世界制品`，包制品协议为 `言域/内容包制品`，格式版本均为 `1`。制品保留规范清单、Schema、原始和已解析内容、本地化、行为或依赖锁、来源索引及统计，并对不含 `摘要` 字段的规范本体计算 SHA-256。

读取和写入都验证协议外形、摘要和语义。写入完成后会重新读取并核对摘要；摘要只能证明字节级完整性，不能代替发行者身份验证或内容审计。

## 公共入口

- `包:言域/内容版本`：语义版本和约束。
- `包:言域/内容包`：清单规范与依赖排序。
- `包:言域/内容文档`：文档规范与定义注册表。
- `包:言域/内容引用`：可见性和引用图。
- `包:言域/内容加载`：文件系统装载与内容图构建。
- `包:言域/内容工具`：检查、制品、差异和文件写入。

Schema 规则见 [CONTENT_SCHEMA.md](CONTENT_SCHEMA.md)，热更新策略见 [HOT_RELOAD.md](HOT_RELOAD.md)。

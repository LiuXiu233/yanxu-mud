# 实体与组件

## 稳定协议

实体由 `稳定编号 + 原型编号 + 组件集合 + 标签 + 修订号` 组成。编号统一采用 `内容包名:类别/标识`，例如 `基础世界:房间/长安南门`。实体公共实现位于 `src/实体/实体.yx`，组件注册表位于 `src/组件/组件注册表.yx`。

组件在实体快照中使用信封：

```yanju
据【
  「版本」：1，
  「数据」：据【「名称」：「精铁长剑」】
】
```

注册表认识的组件会应用默认值、言据 Schema 和逐版本迁移；不认识的组件信封和额外字段会原样往返，保证内容插件的前向兼容。任何组件值都必须由空、逻辑、数、文字、列和文字键据组成，不能把套接字、任务、回调或其他宿主资源塞入状态。

## 修订与变更集

添加/替换/删除组件、修改原型和标签都会把修订号增加一。变更集记录基础修订、结果修订和有序操作；仓储保存时比较基础修订，冲突返回 `YANYU_ENTITY_REVISION_CONFLICT`。应用变更集先复制实体，失败不会修改原对象。

调用 `确认变更()` 后，当前修订成为下一变更集的基础，已确认操作从内存中移除。快照仍包含完整当前状态，与事件日志的事实记录用途不同。

## 原型

原型也是可序列化实体快照。建立动态实体时，动态组件按稳定编号替换原型组件，标签取并集；之后实例状态独立变化。多层原型解析和循环检查由内容运行时负责，实体协议只处理已经解析的一层快照。

## 首批通用组件

`src/组件/通用组件.yx` 注册 22 个题材无关组件：描述、位置、容器、出口、可移动、属性、状态、背包、装备、战斗、技能、冷却、交互、对话、任务、掉落、AI、账户、权限、频道、定时和持久化。它们都拥有稳定编号、v1 Schema、默认值、序列化边界、迁移入口、中文说明和规格测试。

核心组件不能写死修仙境界、门派、职业或具体世界观。游戏需要的额外字段可以由内容 Schema 扩展，题材专属组件应使用游戏自己的命名空间。

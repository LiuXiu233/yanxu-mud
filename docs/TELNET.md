# Telnet 接入

## 启动与连接

青石镇默认只监听回环地址：

```bash
cd examples/青石镇
yanxu 包 运行 . -- --模式 Telnet --地址 127.0.0.1:4000 --存档 .yanxu/telnet-state.json
telnet 127.0.0.1 4000
```

连接后输入 `游客` 建立游客会话，或输入 `登录 <登录名> <密钥>`。成功响应会显示一次恢复令牌；断线后可在会话保留期内输入 `恢复 <令牌>`，成功恢复会轮换令牌，旧令牌立即失效。`会话` 查看当前投影，`退出` 主动关闭会话。其余行按青石镇命令解析。

## 协议行为

`包:言域/Telnet协议` 提供 `Telnet连接状态`、NVT 输出编码和稳定命令/选项常量。初始协商为：

- 服务端 `WILL SUPPRESS-GO-AHEAD`；
- 服务端 `DO SUPPRESS-GO-AHEAD`、`DO NAWS`、`DO CHARSET`；
- 不发送 `WILL ECHO`，客户端保留本地回显；
- 字符集子协商请求 `UTF-8`。

输入解析支持分块 UTF-8、CR LF、CR NUL、LF、多字节字符退格、IAC 转义、NAWS 和字符集子协商。未知远端选项以 `DONT` 拒绝，未知本地选项以 `WONT` 拒绝。输出把换行规范为 NVT CR LF、裸 CR 规范为 CR NUL，并对数据中的 IAC 双写。

## 预算与关闭

默认单行上限 4096 字节，单次读取最多产生 64 行、处理 128 个协商命令和 256 字节子协商。服务器默认每次读取 8192 字节，每连接最多读取 1024 次，输出批次最多 64 项或 262144 字节。输入超限、非法 UTF-8、协商洪泛、子协商畸形、空闲超时、速率限制和输出背压都会产生稳定错误并关闭连接。

`包:言域/Telnet服务器` 的 `Telnet服务器（网关，地址，连接编号生成器，时钟，配置）` 提供 `处理连接`、`运行若干` 和 `运行`。`运行若干` 用于确定性端到端测试；零连接目标表示持续接受。适配器只调用接入网关，不访问世界内部对象。

## 安全边界

Telnet 没有传输加密，不得直接暴露到公网。生产部署必须放在受信网络、VPN 或提供 TLS 的终止代理之后。命令行登录会经过网络明文，优先使用浏览器上的 WSS；日志、错误和审计投影不得包含密码或恢复令牌。一个角色同一时刻只允许一个活动会话控制。

# 结构化消息

世界和命令层只产生结构化消息。`src/消息/消息协议.yx` 定义 `言域消息` v1 节点树，`src/消息/消息渲染.yx` 在接入边界生成纯文本、ANSI Telnet、Web JSON 或言据。

## 节点

v1 提供文字、富文本、人物引用、物品引用、房间引用、命令链接、列表、表格、状态栏、进度条、场景描述、系统提示、错误、频道、战斗和自定义节点。引用使用稳定的 `内容包名:类别/标识`，显示文字独立保存。

```yanxu
定 场景：典 为 消息.场景描述（
    「青石镇南门」，
    【消息.文字（「城门外是一条官道。」）】，
    【消息.房间引用（「青石镇:房间/北街」，「北街」）】
）；
定 信封：典 为 消息.新消息（【场景】，{「语言」：「zh-CN」}）；
```

富文本只接受前景、背景、加粗、下划线和斜体白名单；颜色限定为八种基础色和默认色。数据中不保存 ANSI 转义序列或 HTML。自定义节点使用稳定自定义种类、普通值数据和必需回退节点，旧客户端仍能显示语义。

验证递归检查节点类型、必需字段、表格宽度、进度范围、场景出口和全部普通值边界。单条普通值最多 64 层、10000 个节点，并拒绝循环引用；网络入口还必须在解析前限制帧与字符串字节数。构造器与消息信封都深复制输入，调用方后续修改不会影响已经产生的输出。世界循环按项验证阶段返回的整列消息，全部通过后再原子加入输出队列；聚合列表不误用单条消息预算，非法列表也不会泄漏前面的合法项。

频道服务在消息元数据中写入稳定 `频道编号`、`发送者编号`、可选 `目标编号` 和排序后的 `接收者` 实体编号列。会话与接入层必须只向该列中当前在线且绑定对应角色的会话投递，不能把单个世界输出广播给所有连接。元数据不包含连接对象、会话令牌或网络地址。

## 渲染器

- `纯文本` 输出无 ANSI/HTML 解释的 Unicode 文字。
- `ANSITelnet` 只从样式白名单生成 SGR 序列，并移除内容中的 C0、DEL、C1 控制字符；换行和制表保留。
- `Web值` 返回消息树深副本；`WebJSON` 和 `Web言据` 分别稳定序列化，供 WebSocket 与 HTTP 使用。

Web 客户端必须把文字节点写入 `textContent` 或经过等价转义的文本上下文，不能把内容当作可信 HTML。Telnet 适配器负责编码、协商、窗口和背压；消息渲染器不持有套接字。

## 兼容

未知的内置节点种类在 v1 中拒绝；扩展必须使用自定义节点。增加可选字段可以向后兼容，改变现有字段含义或必需性需要新协议版本。客户端应保留不认识的元数据，并以自定义节点的回退内容显示。

# 内容热重载

热重载运行时把“构建候选世界”和“替换当前世界”分成两个阶段。候选内容必须先完成文件读取、言据解析、Schema、依赖、引用、原型、本地化和行为检查；任何失败都不会触碰当前内容图。

## 流程

```text
读取候选包
  -> 构建候选内容图
  -> 计算确定性差异
  -> 影响分析与静态策略
  -> 现场影响检查
  -> 生成计划
  -> 核对运行时代数
  -> 原子替换当前图
  -> 发布系统事件
  -> 记录审计结果
```

`内容热重载.生成计划（旧图，新图）` 返回协议 `言域/内容热重载计划`。计划包含新旧摘要、差异、传递影响、是否允许和结构化阻止原因。

## 默认允许范围

包的新旧清单都必须声明 `热重载：真`。默认可以评估并应用文本、房间描述、出口、NPC 或物品原型、掉落表、对话、任务文本、本地化和其他非破坏性数据变化。

以下变化默认拒绝：

- 新增或删除运行中的内容包；
- Schema 或行为注册表变化；
- 包依赖、权限或迁移声明变化；
- 组件、存储模型、数据库字段、核心代码或迁移结构变化；
- 删除仍被静态内容引用的定义；
- 现场影响检查器报告仍有活动实例或流程依赖该定义。

删除未被静态引用的定义仍需由现场影响检查器确认没有动态实例。内容包的 `热重载` 标记只表示进入评估，不会绕过安全策略。

## 原子提交

```yanxu
引「包:言域/内容热重载」为 热重载；

定 运行时 为 热重载.运行时（当前图，时钟，影响检查器，事件接收器，审计接收器）；
定 预检：典 为 运行时.预检（候选图）；
若 预检【「允许」】 则
    定 结果：典 为 运行时.提交（候选图，运行时.代数（））；
终
```

每次成功替换都会增加代数。提交必须携带调用方预期的旧代数；不一致返回 `YANYU_CONTENT_RELOAD_CONFLICT`，防止较旧候选覆盖较新的内容。运行时拒绝重入，并在全部成功与失败路径释放忙碌状态。

当前图只在候选图通过全部检查且代数匹配后一次替换。事件或审计接收器在替换后的通知失败不会回滚已经生效的内容图；结果会在 `通知错误` 中记录，并保留本地事件与审计记录供运维补偿。影响检查器失败发生在替换前，会中止提交。

## 审计与安全

运行时为成功和失败保存有界记录，最多保留最近 1000 项。成功事件使用结构化系统事件，携带旧摘要、新摘要、代数和差异统计。回调错误会清理为稳定公开错误，不保存原始凭据、文件内容或源码踪迹。

内容热重载不更新言序代码、原生扩展、数据库 Schema 或权限边界。这些变化必须经过迁移、重启和独立部署审查。

# 状态仓储与世界事务

`src/持久化/仓储协议.yx` 冻结状态仓储能力面，`src/持久化/内存仓储.yx` 是参考实现和契约测试替身。世界内核依赖协议，不依赖 SQLite、文件布局或数据库连接。

## 数据边界

仓储保存动态实体快照、世界变量、计划任务和迁移记录。房间定义、物品/NPC 原型、任务模板、对话和掉落表属于静态内容包，不写入状态仓储。连接、当前命令上下文、输出缓冲、事件处理栈和查询缓存默认不持久化。

所有入口拒绝回调、套接字和其他非普通值；读取返回深副本或从快照恢复的新实体。调用方修改读取结果不会偷偷改变仓储。64 层、10000 节点的复杂度预算作用于单个实体、变量、任务、迁移记录或元数据，而不是整个世界容器；整库状态逐项验证和复制，因此一万个实体不会被误判为单个恶意值。

## 实体和查询

新实体用空预期修订写入；更新完整快照必须给出当前修订，并由新快照严格推进修订。修订、快照序号和迁移序号必须是有限非负整数。变更集由实体协议逐项应用，基础修订不一致返回 `YANYU_REPOSITORY_REVISION_CONFLICT`。删除目标已不存在时，空预期修订是幂等的 `false`；非空预期修订返回冲突，不能把并发删除伪装成普通缺失。

查询可组合编号、编号前缀、原型编号、标签、组件和修订范围，结果按稳定编号排序。非法字段和类型即使在空仓储中也会验证并失败，避免数据量改变输入语义。

## 变量、任务和迁移

世界变量以非空、无首尾空白的名称保存任意普通值。计划任务含编号、到期、有限非负整数间隔、载荷和仓储修订，按到期与编号排序；创建使用空预期修订，更新和删除可做乐观检查。更新前会验证新修订在数值表示中严格大于当前修订；到达浮点整数精度边界时稳定拒绝，旧预期修订不能被重复用于写入。迁移记录按编号去重并保存版本、详情和稳定序号。

## 隔离事务

内存事务从当前世代建立独立的顶层映射分支，实体快照、变量、任务和迁移记录作为已验证的不可变值共享；任何写入都只在分支中替换条目或追加到独立列表，不会修改父仓储记录。提交先比较父世代，再交换分支映射，避免每条命令深复制和重新恢复整个世界。任意直接写入或另一个事务先提交都会使旧事务以 `YANYU_REPOSITORY_TRANSACTION_CONFLICT` 失败。回滚丢弃工作分支，结束后的事务不可复用。外部整库导入与快照恢复仍逐项验证并复制，不信任调用方提供的状态。

`src/世界/世界事务.yx` 在仓储事务外再增加领域发件箱。暂存入口只接受通过领域事件协议或结构化消息协议验证的完整信封，裸节点、裸文字和字段残缺的伪事件会在写状态前拒绝。事件必须是尚未分配全局序号、事务编号与当前世界事务一致的事实事件，意图、系统事件和重放信封不能混入提交输出。事件和结构化消息只在状态成功提交后返回；回滚、持久化故障和并发冲突都会清空发件箱，避免发布一个从未发生的事实。

0.2 的发件箱是内存提交结果，保证单进程提交顺序与失败隔离，但尚不承诺进程崩溃后的耐久投递。文件和 SQLite 适配器必须把状态、事件日志和可靠发件箱写进同一个持久化事务，消费者再以稳定事件编号做幂等投递。

## 快照和测试替身

内存仓储支持命名世界快照、元数据、列表、验证后恢复和重复编号保护。1.0 存档适配器还会增加稳定校验和及介质级原子写入；快照正文不出现在列表结果中。

参考仓储可记录深复制的调用参数，并按操作名注入有限次数的稳定故障，用于系统、事务和仓储契约测试。调用记录默认只保留最近 100 条，可用 `设置调用记录上限` 收紧或设为 0 关闭；内部事务工作副本默认关闭记录，避免长期运行无界积累动态状态。该记录是测试能力，不是生产审计日志；生产日志必须执行敏感字段脱敏。

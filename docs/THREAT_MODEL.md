# 威胁模型

## 资产与攻击者

主要资产是账户身份、角色控制权、恢复令牌、管理令牌、动态世界状态、事件日志、内容制品和服务可用性。攻击者可以控制公网客户端、协议分块、请求顺序、重连频率、内容包文件和显示文字；默认不能修改可信代码、环境变量、代理配置或宿主文件权限。

## 主要威胁

| 威胁 | 控制 | 剩余风险 |
| --- | --- | --- |
| 凭据窃听 | 默认回环；公网必须 TLS/WSS 或 VPN | 原生 Telnet 本身仍是明文 |
| 登录枚举与爆破 | 统一外部失败、输入窗口限流、凭据不回显 | 分布式攻击需代理或账户级限流 |
| 恢复令牌重放 | 只存 HMAC 摘要、成功即轮换、会话到期 | 客户端存储被攻破仍可劫持当前令牌 |
| 跨站 WebSocket 劫持 | 精确 Origin allowlist、缺失 Origin 默认拒绝 | 错误代理改写 Origin 会破坏边界 |
| 帧/行/首部内存耗尽 | 分层字节、项目、读取与队列上限 | 大量并发连接需宿主级连接上限 |
| 输出背压拖垮 | 每会话有界队列，过载关闭 | 生产宿主仍需公平调度 |
| 静态路径穿越 | 拒绝绝对路径、危险段、反斜杠和 NUL | 文件根配置错误仍可暴露不应公开文件 |
| XSS | 外部脚本禁止、CSP、结构化 DOM `textContent` 渲染 | 浏览器依赖或代理注入需独立审计 |
| 管理接口滥用 | 独立 Bearer、缺失默认拒绝、恒时比较 | 单令牌轮换和操作级授权由宿主完善 |
| 内容包执行代码 | 言据不执行代码，只引用注册行为 | 代码插件和原生扩展仍是完全可信代码 |
| 存档篡改或回滚 | 校验和、世代链、快照预检和恢复验证 | 青石镇轻量 JSON 仅用于示例，不防恶意本地写入 |
| 并发写入破坏 | 事务冲突与数据库约束 | 轻量存档无跨进程锁，必须单写者 |
| 日志泄密 | 秘密字段禁止进入投影和错误，管理查询需授权 | 宿主或代理原始访问日志仍需脱敏 |

## 验证

安全规格覆盖秘密不回显、审计无令牌、速率限制、背压、会话角色独占、恢复轮换、Telnet 协商洪泛、WebSocket Origin/掩码/UTF-8/大小限制、HTTP 管理拒绝、CSP 和静态路径穿越。`scripts/network-e2e.ps1` 在真实回环套接字上验证三个传输，CI 在 Windows、Linux 和 macOS 运行。

新增协议字段、网络权限、环境变量、原生扩展、管理操作或静态文件根时，必须更新此模型和相应失败测试后才能发布。

# 系统与调度

系统注册表位于 `src/系统/系统注册表.yx`。系统以稳定编号注册，把规则表达为接收上下文的处理器；实体只保存状态，不持有长期运行任务。

## 生命周期

一个系统具有编号、优先级、处理器、可选启动器和可选停止器。注册表支持添加、替换、禁用、启用、装饰、排序、查询以及批量启动、处理和停止。

```yanxu
定 移动 为 系统.系统（
    「言域:系统/移动」，
    100，
    处理移动，
    初始化移动，
    停止移动
）；
注册表.注册（移动）；
```

优先级必须是有限整数；数值较小者先处理，相同优先级按稳定编号排序。启动按正序执行，停止按逆序执行，便于后启动的依赖先释放。装饰器接收旧处理器并返回新处理器，可用于追踪、权限或指标，而无需修改规则实现。

系统必须启动后才能处理；停止是终态，防止已经释放资源的实例被误复用。需要重启时应建立并替换一个新实例。

`启动全部` 具有失败清理语义：任一启动器失败时，注册表先清理失败候选，再按逆序停止本次已经启动的实例，并保持整体未运行；尚未轮到的实例不会被启动。由于已清理实例进入“已停止”终态，调用方应修正故障后用新实例建立新的注册表，而不能把半次启动当作可继续运行的世界。清理器异常会作为脱敏后的结构化详情随生命周期错误返回，并且不会阻止其余实例继续清理。

处理器必须是函数，启动器和停止器必须为空或函数；构造时立即验证这些边界。装饰器自身及其产物也都必须是函数，装饰失败不会改写原处理器。这些配置、启动、停止和装饰错误统一使用 `YANYU_SYSTEM_LIFECYCLE`，对外错误详情不会携带回调原始异常消息或源码位置。

## 运行中注册与替换

注册表在 `启动全部` 成功后保存本轮生命周期上下文。此后注册新编号时，新实例会先用该上下文启动，启动成功后才加入注册表；启动失败时映射保持不变，并调用候选停止器清理可能已分配的资源。

`替换` 只接收状态为“已创建”的候选实例，并继承旧实例的启用状态。若注册表正在运行，替换按以下提交顺序执行：

1. 启动候选实例；若失败，清理候选并保留仍在运行的旧实例。
2. 停止旧实例；若失败，停止候选作为回滚，并继续保留旧映射。
3. 两步均成功后才把编号映射切换为候选实例。

因此成功替换不会遗留旧实例资源，失败替换也不会把注册表切换到未启动或已停止的实例。候选启动与旧实例停止之间存在很短的重叠窗口，生命周期回调必须允许同一系统编号的新旧版本短暂并存。若停止器自身失败，注册表会保持原映射和原状态以便重试；回调若已经执行了部分外部副作用，应自行提供幂等清理语义。

对未注册编号调用 `替换` 等同于 `注册`。注册表整体未运行时，替换只切换“已创建”候选，不触发生命周期回调；下一次 `启动全部` 再启动它。

不要绕过注册表直接启动已注册实例。若旧实例已在注册表生命周期之外运行，注册表没有可靠的停止上下文，`替换` 会拒绝切换，调用方必须先用原上下文显式停止旧实例；这条约束防止无上下文替换遗留旧资源。

## 可注入时间与计划任务

`src/测试支持/虚拟时钟.yx` 提供确定性虚拟时钟和调度器。调度器保存普通值载荷，按到期时间和任务编号排序，支持一次性任务、重复任务、取消、预算、快照与恢复。

生产世界把现实或单调时钟适配到同一“当前毫秒”调用边界；测试直接推进虚拟时钟。冷却、刷新、任务重置、NPC 行为、战斗回合、世界时间和公告都应登记计划任务，不为每个实体创建常驻任务，也不以 30/60 帧空转轮询。

## 系统编写约束

- 通过上下文访问仓储、事务、事件和结构化消息，不直接访问套接字或数据库实现。
- 所有可持久状态写入实体组件、世界变量或计划任务，不藏在不可序列化的闭包中。
- 随机规则接收显式随机源或种子，确保测试和重放可复现。
- 一次处理遵守世界轮次预算；长工作拆成可恢复的计划任务。

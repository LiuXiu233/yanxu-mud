# 系统与调度

系统注册表位于 `src/系统/系统注册表.yx`。系统以稳定编号注册，把规则表达为接收上下文的处理器；实体只保存状态，不持有长期运行任务。

## 生命周期

一个系统具有编号、优先级、处理器、可选启动器和可选停止器。注册表支持添加、替换、禁用、启用、装饰、排序、查询以及批量启动、处理和停止。

```yanxu
定 移动 为 系统.系统（
    「言域:系统/移动」，
    100，
    处理移动，
    初始化移动，
    停止移动
）；
注册表.注册（移动）；
```

优先级数值较小者先处理，相同优先级按稳定编号排序。启动按正序执行，停止按逆序执行，便于后启动的依赖先释放。装饰器接收旧处理器并返回新处理器，可用于追踪、权限或指标，而无需修改规则实现。

系统必须启动后才能处理；停止是终态，防止已经释放资源的实例被误复用。需要重启时应建立并替换一个新实例。

## 可注入时间与计划任务

`src/测试支持/虚拟时钟.yx` 提供确定性虚拟时钟和调度器。调度器保存普通值载荷，按到期时间和任务编号排序，支持一次性任务、重复任务、取消、预算、快照与恢复。

生产世界把现实或单调时钟适配到同一“当前毫秒”调用边界；测试直接推进虚拟时钟。冷却、刷新、任务重置、NPC 行为、战斗回合、世界时间和公告都应登记计划任务，不为每个实体创建常驻任务，也不以 30/60 帧空转轮询。

## 系统编写约束

- 通过上下文访问仓储、事务、事件和结构化消息，不直接访问套接字或数据库实现。
- 所有可持久状态写入实体组件、世界变量或计划任务，不藏在不可序列化的闭包中。
- 随机规则接收显式随机源或种子，确保测试和重放可复现。
- 一次处理遵守世界轮次预算；长工作拆成可恢复的计划任务。

# WebSocket 接入

## 启动

```bash
cd examples/青石镇
yanxu 包 运行 . -- --模式 WebSocket --地址 127.0.0.1:8081 --存档 .yanxu/websocket-state.json
```

入口为 `ws://127.0.0.1:8081/ws`。青石镇默认允许来自 `http://127.0.0.1:8080` 和 `http://localhost:8080` 的浏览器 Origin，拒绝缺失或不在列表中的 Origin。嵌入游戏应显式提供自己的来源列表；不能用反射请求头或通配来源代替配置。

## 握手与帧

`包:言域/WebSocket协议` 实现 RFC 6455 HTTP 升级、SHA-1 接受值、Base64、客户端掩码、分片文本、Ping/Pong 和关闭帧。默认只接受 `/ws`、版本 13、无正文 GET；`yanyu.v1` 子协议可选。HTTP 首部读取有 16384 字节上限，并保留与升级请求同批到达的首帧字节。

默认帧上限 65536 字节，完整消息上限 262144 字节，单次输入最多解析 64 帧。客户端帧必须掩码；二进制帧默认拒绝；文本和关闭原因必须是有效 UTF-8；控制帧必须完整且不超过 125 字节。协议错误使用合适的 1002、1007、1009 或 1011 关闭码。

## 请求协议

每条客户端文本帧是一个 JSON 对象，至少包含 `种类` 与 `请求编号`。建议先协商能力：

```json
{"种类":"协商","请求编号":"req-1","能力":["结构化消息-v1","请求编号","会话恢复"]}
```

登录、游客与恢复：

```json
{"种类":"登录","请求编号":"req-2","登录名":"player","密码":"secret","角色编号":"青石镇:角色/旅人"}
{"种类":"游客","请求编号":"req-3"}
{"种类":"恢复","请求编号":"req-4","恢复令牌":"one-time-token"}
```

命令、状态与退出：

```json
{"种类":"命令","请求编号":"req-5","命令":"观察"}
{"种类":"状态","请求编号":"req-6"}
{"种类":"退出","请求编号":"req-7"}
```

响应固定包含协议、版本、请求种类、请求编号、成功标记、数据、结构化错误和结构化消息列。登录失败对不存在账户、错误名称或错误密钥只暴露统一消息，不回显凭据。成功登录、游客进入和恢复返回新的恢复令牌；客户端只应保存最新令牌。

## 服务器与背压

`包:言域/WebSocket服务器` 的 `WebSocket服务器（网关，地址，连接编号生成器，时钟，配置）` 提供 `处理连接`、`运行若干` 和 `运行`。输出先进入会话的有界队列，再按项目数和字节数成批发送；队列过载以 1011 关闭。读取次数、读写超时、输入速率和会话容量均有独立上限。

青石镇浏览器使用 `textContent` 构造节点，不把服务端文字解释为 HTML。公网部署必须由反向代理提供 TLS，客户端地址使用 `wss://`，并在代理与应用两层限制首部、消息、连接速率和空闲时间。

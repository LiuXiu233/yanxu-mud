# 部署

## 进程拓扑

言序 1.1.12 的任务是单线程协作式任务。青石镇将接入拆为独立进程：

```text
浏览器 ──HTTPS── 反向代理 ──HTTP── HTTP 静态/管理进程
   └────WSS────── 反向代理 ──WS──── WebSocket 玩法进程
终端 ───TLS/VPN──────────────────── Telnet 玩法进程
```

不要声称一个青石镇进程同时运行三个独立监听器。HTTP 进程不持有 WebSocket 进程的会话；浏览器直接在 WebSocket 上认证。示例轻量存档没有跨进程锁，同一存档路径只能交给一个可写玩法进程。HTTP、WebSocket 和 Telnet 示例应使用各自路径，或由部署宿主提供真正共享且有并发控制的仓储。

## 最小配置

所有示例监听地址默认为回环。生产至少配置：

- 32 字节以上、随机且持久保存的 `YANYU_SESSION_SECRET`；
- 需要示例登录时同时配置 `YANYU_LOGIN_NAME` 与 `YANYU_LOGIN_SECRET`；
- 启用管理端点时配置独立 `YANYU_ADMIN_TOKEN`；
- 反向代理上的 TLS、HSTS、请求体/首部上限、连接速率和空闲超时；
- 与公开站点完全一致的 WebSocket Origin allowlist；
- 仅运行账户可读的环境和存档目录权限。

青石镇环境登录只是演示适配器，不是多账户密码数据库。实际服务应将接入网关连接到 `账户服务` 和成熟的 Argon2id、scrypt 或 bcrypt 适配器，不得把 SHA-256 当作密码存储算法。

## 启动与检查

部署前在默认预算运行：

```bash
pwsh -NoProfile -File scripts/network-e2e.ps1 -Yanxu "$(command -v yanxu)"
```

Windows 可传入 `(Get-Command yanxu).Source`。完整验证脚本还会运行静态检查、单元/集成规格、双执行器兼容、API 消费者契约、青石镇独立规格、内容重建和发行构建。

健康探针使用 `GET /healthz`。就绪检查还应验证 `GET /api/v1/server`、内容摘要、玩法进程端口和反向代理的 WSS 握手。日志采集必须在落盘前删除密码、登录密钥、恢复令牌、Cookie、Authorization、数据库凭据和私钥。

## 更新与恢复

1. 备份轻量存档或创建生产仓储快照。
2. 在隔离目录运行内容检查、内容差异和迁移计划。
3. 停止唯一可写玩法进程。
4. 替换已校验的代码和内容制品，运行迁移。
5. 启动并核对版本、内容摘要、快照和事件日志。
6. 恢复代理流量；失败时停止新进程并从已验证快照恢复。

0.x 不承诺生产兼容。升级到后续预览版前必须阅读变更记录；1.0 后按 `COMPATIBILITY.md` 的稳定面执行滚动升级。

# 领域事件

事件总线位于 `src/事件/事件总线.yx`。它只处理普通言序值和回调，不依赖网络、数据库或任何渲染格式。

## 事件信封

`新事件` 建立一个稳定信封，字段包括事件编号、全局序号、世界时间、现实时间、类别、事件种类、来源、目标、载荷、命令编号、事务编号、追踪编号和内容包版本。类别只能是 `意图`、`事实` 或 `系统`；事件种类使用 `内容包名:类别/标识` 格式。

新事件的全局序号为零，第一次进入总线时分配严格递增的序号。重放保留原序号，并推进总线的下一序号，避免随后发布的事件与历史事件冲突。载荷在入队、分发、历史查询和回调边界均深复制，处理器不能借可变引用篡改日志。

## 订阅与顺序

```yanxu
定 总线 为 事件.总线（16，10000）；
定 订阅编号：文 为 总线.订阅（
    「青石镇:事件/进入房间」，
    处理进入，
    100，
    只看玩家，
    假
）；
```

优先级数值较小者先执行；相同优先级严格按注册顺序执行。空事件种类表示订阅全部事件。过滤器失败只跳过当前订阅并写入踪迹，不会破坏其他订阅。单次订阅在第一次匹配并调用后自动移除，无论处理器成功或返回结构化错误。

## 同步、队列与重放

- `发布` 同步调用匹配的处理器，返回每个处理器的结构化结果。
- `入队` 保存事件副本；`排空` 按先进先出顺序处理指定数量。
- `重放` 接受历史事件列，保留协议字段并附加重放标记。
- `历史` 返回已经开始分发的事件副本；`踪迹` 返回本批次的结构化执行阶段。

每个顶层发布批次有最大事件预算，嵌套发布共享该预算。递归深度也独立受限。超过边界分别使用 `YANYU_EVENT_BUDGET_EXCEEDED` 和 `YANYU_EVENT_RECURSION_LIMIT`，防止插件或系统形成无限事件链。

## 事务边界

命令和系统应先在世界事务中暂存事实事件。只有状态变更成功提交后才把事件交给总线；回滚事务不得泄漏事实事件、消息或持久化副作用。事件日志记录提交后的重要事实，不把每个内部意图都当作完整事件溯源数据。

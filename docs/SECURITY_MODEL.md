# 安全模型

## 信任边界

网络字节、HTTP 首部、JSON、Telnet 行、命令参数和言据内容包均不受信任。接入层负责协议解析、长度预算、速率限制、认证和会话绑定；世界层只接收已经规范化的命令上下文，不读取套接字或客户端权限声明。

代码插件、原生扩展、密码哈希适配器、数据库驱动和部署代理属于可信计算基。内容插件只含言据和稳定行为引用，不执行代码。浏览器消息文字仍按不可信数据处理，只能写入 `textContent` 或等价安全上下文。

## 身份链

```text
TCP 连接 -> 会话 -> 账户 -> 角色 -> 世界实体
```

连接断开不会自动删除会话。恢复令牌由安全随机生成，只保存 HMAC-SHA-256 摘要，成功恢复后轮换。一个活动角色只允许一个会话控制。公开投影不含令牌摘要、密码、登录密钥或管理令牌。

青石镇演示登录从环境读取名称与密钥，使用固定长度摘要恒时比较，不持久保存凭据。缺少任一变量时登录关闭，游客模式仍可按配置使用。多账户生产服务必须使用账户服务和成熟密码哈希实现。

## 网络控制

- Telnet、WebSocket、HTTP 首部、帧、消息、行、读取次数和队列均有硬上限。
- WebSocket 要求客户端掩码和显式 Origin allowlist，拒绝二进制与非法 UTF-8。
- HTTP 静态路径拒绝穿越，响应启用 CSP、`nosniff`、Referrer Policy 和 `no-store`。
- 管理接口只接受 Authorization Bearer；未配置令牌时默认拒绝。
- 默认只监听回环地址；TLS、HSTS 和公网连接治理由受信代理提供。

## 数据控制

仓储使用乐观修订、事务、快照校验和和恢复后摘要验证。静态内容不进入动态仓储。事件日志使用校验和链；日志查询返回普通值副本。跨进程写入必须由具备锁或事务隔离的生产仓储协调，青石镇轻量存档不支持并发写者。

结构化错误对外只返回稳定代码、中文消息和受控详情。内部错误踪迹可以进入受保护诊断通道，但必须先脱敏。安全事件包括登录拒绝、速率限制、会话恢复拒绝、角色占用、管理拒绝、协议关闭、热重载拒绝和快照操作。
